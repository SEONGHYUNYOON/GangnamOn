# 강남온(GangnamOn) 현황 분석 & 디자인·메뉴 개선 제안

작성일: 2026-07-06

---

## 1. 현재까지 작업된 내용 (파악 결과)

먼저 좋은 소식: 말씀하신 기능들은 **이미 대부분 코드로 구현되어 있습니다.** "새로 만들어야 하는" 단계가 아니라 "다듬고 활성화시키는" 단계예요.

기술 스택: React + Vite + Tailwind CSS, DB는 Supabase, 배포는 Vercel. 원래 "파주온(Paju On)"이라는 파주 지역 커뮤니티로 시작했다가 강남으로 리브랜딩한 이력이 git 기록에 남아있고, 지금 디자인이 톤앤매너별로 제각각인 느낌이 드는 원인 중 하나가 여기 있는 것으로 보입니다.

이미 구현된 메뉴/기능:

- **자영업자 이벤트 홍보** → `Owner's Note` 탭 (`posts` 테이블의 `type='event'`). 인스타그램 피드 스타일로 사진+카피+마감 타이머+쿠폰 다운로드 버튼까지 있음.
- **주민 커플 매칭(틴더 스타일)** → `강남 썸&쌈` (GangnamRomance.jsx). 스와이프 UI, 매너온도(`manner_temp`) 필드까지 DB에 준비돼 있음.
- **게임 기록 경쟁** → `강남 라운지` (GangnamLounge.jsx) 안에 스네이크, 벽돌깨기, 두더지잡기, 타자연습, 반응속도, 타워디펜스 등 게임 8종 + 랭킹 로직(`gameRank.js`)까지 이미 구축됨.
- **혼밥 매칭** → `DiningCompanion.jsx` 존재.
- **아이러브스쿨식 동문회** → `ILoveSchool.jsx`, 사이드바에 "학교 찾기&동창회 / 친구 찾기" 메뉴로 연결됨.
- **중고 장터** → `UsedMarket.jsx`, `posts.type='market'`로 연동.
- **플로우 배너** → 홈 상단 마퀴(흐르는 글자) 배너 + 500온으로 직접 등록하는 `BannerWriteModal.jsx`까지 구현됨.
- 그 외에도 미니홈피(싸이월드 감성), 아바타 커스터마이징, 리워드센터(온=가상재화), 관리자 대시보드, 카카오맵 연동, 문화클래스, 채팅위젯 등 상당히 많은 기능이 이미 존재.

즉, "메뉴를 활성화하고 싶다"는 요청은 **없는 걸 새로 만드는 것보다, 이미 만든 기능을 사람들이 실제로 쓰게 만드는 문제**(노출·동선·완성도)에 더 가깝습니다.

### 정리가 필요한 부분

- 루트에 `src/`(현재 메인)와 `gangnamon/`(예전 버전으로 추정되는 중복 폴더)가 같이 있어 혼동 소지가 있습니다. 나중에 `gangnamon/` 폴더는 백업 후 정리하는 걸 추천드려요.
- 예전 코드 검수 보고서(`docs/CODE_REVIEW_REPORT.md`)에 이미 발견된 사소한 오탈자/아이콘 불일치(콩 이모지 🫘가 "온" 화폐로 바뀐 뒤에도 일부 남아있는 것 등)가 아직 반영 안 된 상태입니다.

## 2. 디자인이 "촌스러워" 보이는 이유

화면마다 톤앤매너가 다르게 섞여 있는 게 가장 큰 원인입니다.

- 홈/전체: 보라(purple)-핑크 그라데이션 버튼
- 강남 라운지: 남색 아케이드 다크 테마
- 강남 썸&쌈: 어두운 버건디/로즈 톤
- Owner's Note: 아이보리 배경 + 세리프 이탤릭체의 "매거진st" 감성

네 가지 스타일이 앱 하나 안에서 충돌하고 있어서, 각 화면은 나쁘지 않은데 전체적으로는 "이것저것 갖다 붙인" 인상을 줍니다. 여기에 더해 구조적으로도:

- 버튼/배지마다 그라데이션 남발 (강조가 너무 많으면 강조가 없는 것과 같아짐)
- 이모지가 UI 라벨(제목, 버튼)에도 아이콘 대신 쓰이는 경우가 많음 — 정작 아이콘 라이브러리(lucide-react)는 이미 깔려 있음
- 카드 모서리 둥글기가 `rounded-sm`, `rounded-xl`, `rounded-2xl`, `rounded-3xl`이 화면마다 제각각
- 그림자도 색깔 있는 그림자(`shadow-purple-200` 등)를 여기저기 써서 화면이 산만함
- 왼쪽 사이드바는 그룹 7개가 기본적으로 전부 펼쳐진 상태라 글자 크기 작은 메뉴가 화면을 꽉 채움

### 개선 방향 (재작업 아니고 "통일" 작업)

1. **메인 컬러 하나로 통일**: 로고 색(인디고~보라 계열) 하나를 브랜드 컬러로 확정하고, 그라데이션은 "가장 중요한 버튼 1개"에만 사용. 나머지는 단색 + 톤온톤.
2. **다크 테마는 의도적으로 쓰되 컴포넌트 규칙은 공유**: 라운지·로맨스처럼 다크 테마가 어울리는 화면은 유지해도 좋지만, 버튼 모양·모서리 둥글기·그림자 강도는 라이트 테마와 같은 규칙을 따르게 해서 "같은 앱"처럼 보이게.
3. **이모지는 사용자 콘텐츠(배너 문구, 게시글)에만, UI 구조는 아이콘으로**: 제목/버튼/탭 라벨은 lucide 아이콘으로 통일.
4. **모서리 둥글기 2종류로 통일**: 버튼/인풋은 하나, 카드는 하나로 축소.
5. **그림자 1종으로 통일**: 은은한 회색 그림자 하나만 기본으로 쓰고, 컬러 그림자는 정말 강조가 필요한 곳 1~2군데만.
6. **사이드바 접기 기본값 변경**: 현재 탭이 속한 그룹만 펼치고 나머지는 접어서 첫 화면을 가볍게.
7. **참고할 만한 국내 레퍼런스**: 당근마켓(따뜻한 오렌지+미니멀), 에브리타임(깔끔한 화이트+블루 포인트), 여성시대/베이글코드류 커뮤니티 앱의 "카드형 피드" 등. 강남온만의 정체성을 하나 정해서 그 톤을 전체에 적용하는 걸 추천드려요.

## 3. 메뉴별 강화 아이디어 (이미 있는 기능 기준)

### 자영업자 이벤트 홍보 (Owner's Note)
- 사장님 인증 배지(사업자 인증) 추가 → 신뢰도, "가짜 이벤트" 방지
- "마감 임박순" 정렬 + 지도에서 진행 중인 이벤트 한눈에 보기(카카오맵 이미 연동돼 있음)
- 상단 고정 노출 등 **유료 광고 슬롯 상품화** → 자영업자 유입/수익화 포인트로 바로 연결 가능
- 이벤트 조회수·혜택받기 클릭수를 사장님에게 리포트로 보여주면 재구매(재홍보) 유도

### 커플 매칭 (강남 썸&쌈)
- 실명/직장 인증 또는 본인인증 단계 추가 (안전 이슈가 큰 기능이라 우선순위 높음)
- DB에 이미 있는 `manner_temp`(매너온도)를 프로필 카드에 노출해 신뢰 신호로 활용
- 매칭 후 대화 유효시간(예: 48시간) 두어 "지금 답장 안 하면 사라짐" 긴장감 부여
- 신고/차단 기능 (안전장치는 필수)

### 게임 경쟁 (강남 라운지)
- 이미 게임이 8종이나 있는 게 강점 → 주간/월간 랭킹 초기화 + 1~3등 온(재화) 보상 지급
- 홈 화면 우측 패널에 "이번 주 랭킹 TOP3" 위젯을 노출해서 라운지로 트래픽 유도
- 여러 게임 점수를 합산한 "강남 게이머 등급" 배지 도입

### 혼밥 매칭 (DiningCompanion)
- "오늘 점심/오늘 저녁" 같은 시간대 기반 매칭으로 즉시성 강화
- Owner's Note 이벤트와 연계: "혼밥 매칭 성사 시 근처 할인 매장 추천" 식으로 두 메뉴를 교차 활용
- 매너온도 노출로 안전감 부여

### 아이러브스쿨 동문회 (ILoveSchool)
- 학교+졸업연도 기반 자동 매칭 알림 ("같은 학교 동문이 가입했어요")
- 기수별 게시판/그룹 개설 기능
- 옛날 아이러브스쿨 특유의 "이 사람 나 기억해?" 감성 태그 기능으로 향수 자극

### 중고 장터 (UsedMarket)
- 이미 있는 채팅위젯(ChatWidget)과 상품 문의 버튼 연결 (지금은 분리돼 있을 가능성)
- 카테고리 필터, 거래완료 처리 상태 표시
- 동네 인증 기반으로 "직거래 안전지역" 신뢰 강화

### 플로우 배너
- 지금은 텍스트만 흘러가는 구조 → **클릭하면 해당 이벤트/모임/상품 상세로 이동**하도록 연결해서 "사용자 목적성 강화" 요청과 직접 연결
- 배너 유형(공지/이벤트/개인글)별 색상 태그 구분
- 유료 등록(500온) 구조는 이미 좋음 → 노출 위치/기간 차등 상품화 가능

## 4. 다음 단계 제안

1순위: 디자인 시스템 통일(컬러/모서리/그림자/아이콘 규칙 확정) → 전체 화면에 일괄 적용
2순위: 플로우 배너 클릭 시 이동 기능(목적성 강화) — 작업량 대비 효과 큼
3순위: 안전장치(매칭 신고/인증) — 커플 매칭, 혼밥 매칭처럼 실제 만남으로 이어지는 기능에 필수
4순위: 자영업자 이벤트 유료 광고 상품화 — 수익화 + 사용자 유입 동시 해결

원하시면 이 중 하나를 골라서 구체적인 화면 시안이나 실제 코드 수정으로 진행해드릴게요.
