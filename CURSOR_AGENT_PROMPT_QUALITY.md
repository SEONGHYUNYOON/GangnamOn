# Cursor AI 에이전트용 프롬프트 (비밀번호 찾기 / 약관 동의 배포)

아래 내용을 그대로 Cursor의 AI 에이전트(Agent 모드)에 붙여넣어 주세요.
현재 폴더(D:\DEV\GangNam_On)를 열어놓은 상태에서 실행하시면 됩니다.

이번 작업은 Appwrite 스키마나 Function 변경이 없는 **순수 프론트엔드 변경**이라
지난번보다 훨씬 간단합니다. git push만 하면 Vercel이 알아서 재배포합니다.

---

## 프롬프트 (복사해서 붙여넣기)

```
너는 GangnamOn 프로젝트(D:\DEV\GangNam_On)에 추가된 아래 기능들을
검증하고 배포해야 해.

## 이번에 추가된 것 (전부 코드 작성 완료, 로컬 빌드 통과 확인함)
1. 비밀번호 찾기: AuthWidget.jsx에 "비밀번호를 잊으셨나요?" 링크 추가.
   Appwrite의 account.createRecovery()로 재설정 메일을 보내고,
   메일의 링크를 클릭하면 (?flow=recovery&userId=...&secret=... 파라미터로
   돌아옴) App.jsx가 이를 감지해서 새 컴포넌트 ResetPasswordModal.jsx를
   띄워 새 비밀번호를 입력받고 account.updateRecovery()로 반영해.
2. 이용약관 / 개인정보처리방침: TermsAndPrivacyModal.jsx라는 새 컴포넌트로
   약관 전문을 보여주고, 회원가입 폼에 필수 동의 체크박스를 추가했어
   (동의 안 하면 가입이 막힘). 좌측 사이드바 하단에도 약관을 열어볼 수
   있는 링크를 추가했어.
3. 버그 수정: 회원가입 시 프로필 문서에 본인 update/delete 권한이 빠져있던
   문제를 고쳤어 (이게 있으면 아바타 변경, 자기소개 수정 등이 새 가입자만
   실패하는 버그가 있었음).

## 네가 해야 할 일 (순서대로)

1. `npm run build` 실행해서 에러 없는지 확인.

2. `npm run dev`로 로컬 서버 띄우고 직접 테스트:
   - 새로 회원가입 시도 → 약관 체크박스 체크 안 하면 가입이 막히는지,
     체크하면 정상 가입되는지
   - 로그인 화면에서 "비밀번호를 잊으셨나요?" 클릭 → 이메일 입력 →
     실제로 재설정 메일이 오는지 (본인 이메일로 테스트해줘)
   - 메일의 링크를 클릭 → 새 비밀번호 입력 화면이 뜨는지 → 새 비밀번호로
     로그인이 되는지
   - 좌측 사이드바 하단 "이용약관" / "개인정보처리방침" 링크가 모달을
     여는지
   - 회원가입 직후 아바타 변경이 정상 동작하는지 (권한 버그 수정 확인용)
   - 에러가 나면 원인 파악해서 고쳐줘.

3. 로컬 테스트가 전부 정상이면 git에 커밋 & 푸시:
   - `git add -A`
   - `git commit -m "Add password recovery, terms consent, fix profile permission bug"`
   - `git push`

4. Vercel 자동 배포 완료 후, 실제 사이트(https://gangnam-on.vercel.app)에서
   2번 테스트를 다시 한번 반복해줘.

5. 결과를 나에게 요약해서 알려줘: 정상 동작한 것, 문제 있었던 것과 어떻게
   고쳤는지.

## 하지 말아야 할 것
- Appwrite CLI로 스키마나 Function을 다시 push할 필요 없어 (이번엔 DB/
  Function 변경이 전혀 없음). 괜히 `appwrite push` 계열 명령어를 실행하지 마.
- 이용약관/개인정보처리방침 문구 내용을 임의로 크게 바꾸지 마 (법적 문구라
  신중하게 다뤄야 함). 오탈자 수준의 수정은 괜찮아.
```
